# Task Specification: T21 - Sprint 1

## Objective
Refactor `src/sprint/ci-status-utils.ts` to remove the redundant `isCiCheckFailed` and `isCiRunFailed` predicates and replace them with a single, unified source of truth.

## Files to Modify
- `src/sprint/ci-status-utils.ts`: De-duplicate the status predicate logic.
- `src/domain/sprint/ci/feature-pr-gate.ts`: Update callers.
- `src/domain/sprint/ci/main-merge-gate.ts`: Update callers.

## Technical Details & Research Findings
- **Context**: `isCiCheckFailed` and `isCiRunFailed` have identical logic for determining if a CI check or run has failed based on its status and conclusion.
- **Improvements**:
    - Implement a single `isCiFailure(status: string, conclusion: string | null): boolean` function.
    - Implement a single `isCiPending(status: string, conclusion: string | null): boolean` function.
    - Export these as the standard way to evaluate CI results.
- **Constraints**: Ensure the de-duplicated logic handles all current conclusion variants (`success`, `neutral`, `skipped`).

## Execution Steps
1. Create a unified `isCiFailure` function in `src/sprint/ci-status-utils.ts`.
2. Refactor existing code to use the new unified predicates.
3. Remove the redundant functions once all call sites are migrated.
4. Verify by running tests in `tests/backend/sprint/ci-status-utils.test.ts`.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/sprint/ci-status-utils.test.ts`.
- **Criteria**: CI failure and pending status are correctly identified across the entire codebase using a single set of rules.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
