# Task Specification: T19 - Sprint 1

## Objective
Implement a robust, structured parser for subtask markdown files in `SubtaskRepository` to replace the current fragile regex-based extraction.

## Files to Modify
- `src/repositories/subtask-repository.ts`: Refactor `loadSubtasks` and `parseDependsOn`.
- `src/infrastructure/repositories/subtask-file-repository.ts`: Utilize the new parser.

## Technical Details & Research Findings
- **Context**: The `SubtaskRepository` uses basic regexes to extract metadata (title, depends_on, etc.), which can fail on common markdown formatting variants (e.g., different indentation, multiline prompts, quoting).
- **Parser Design**:
    - Build a deterministic frontmatter-like parser that correctly identifies key-value pairs at the beginning of the file.
    - Handle the `prompt:` section as the remaining content after the metadata block.
    - Specifically handle array parsing for `depends_on` with support for optional quotes and spaces.
- **Constraints**: Maintain compatibility with the existing subtask markdown format.

## Execution Steps
1. Implement a specialized `SubtaskParser` class or utility function.
2. Refactor `SubtaskRepository.loadSubtasks` to use the new parser.
3. Replace regex-based `parseDependsOn` with a more reliable implementation.
4. Add comprehensive unit tests for the parser in `tests/backend/repositories/subtask-repository.test.ts`, including edge cases like empty fields, multiline strings, and various array formats.
5. Ensure the `is_independent` and `merged` flags are correctly interpreted as booleans.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/repositories/subtask-repository.test.ts`.
- **Criteria**: The parser correctly extracts all metadata fields from a wide variety of subtask markdown files.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
