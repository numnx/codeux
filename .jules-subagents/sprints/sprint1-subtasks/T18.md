# Task Specification: T18 - Sprint 1

## Objective
Implement `SubtaskFileRepository` to centralize the logic for reading and updating subtask markdown files, and migrate `SprintOrchestrator` and `JulesAgentServer` to use it.

## Files to Modify
- `src/infrastructure/repositories/subtask-file-repository.ts`: Create this new file.
- `src/sprint/sprint-orchestrator.ts`: Refactor to use the new repository.
- `src/server/jules-agent-server.ts`: Refactor to use the new repository.

## Technical Details & Research Findings
- **Context**: `persistTaskMergedFlag` logic is currently duplicated in `SprintOrchestrator.ts` and `JulesAgentServer.ts`, which can lead to inconsistent file mutations.
- **Repository Responsibilities**:
    - `setMerged(taskId: string, merged: boolean)`: Atomically updates the `merged: true/false` flag in the markdown file.
    - `loadSubtask(taskId: string): Promise<Subtask>`: Reads and parses a single subtask file.
    - Handle directory and path resolution for subtasks consistently.
- **Constraints**: Maintain the existing logic for inserting the `merged` flag if it doesn't exist (above the `prompt:` section).

## Execution Steps
1. Create `src/infrastructure/repositories/subtask-file-repository.ts`.
2. Move the `persistTaskMergedFlag` logic to the new repository and generalize it.
3. Update `SprintOrchestrator` and `JulesAgentServer` to use the repository for all subtask file updates.
4. Add unit tests for `SubtaskFileRepository` to ensure it correctly handles various markdown structures (with/without existing merged flag).

## Verification Requirements
