# Task Specification: T13 - Sprint 1

## Objective
Refactor `src/mcp/core-tool-handler.ts` by extracting the summarization and formatting logic into a dedicated domain service `src/domain/sessions/activity-summary.ts`.

## Files to Modify
- `src/domain/sessions/activity-summary.ts`: Create this new file.
- `src/mcp/core-tool-handler.ts`: Refactor to use the new summary service.

## Technical Details & Research Findings
- **Context**: `CoreToolHandler` is currently responsible for both coordinating MCP tool calls and defining the presentation logic (summarization) for sessions, activities, and sources.
- **Decomposition Target**:
    - Extract `toActivitySummary`, `toSessionSummary`, `toActivityCollectionSummary`, `toActivityPageSummary`, `toSourceSummary`, and `toSourcePageSummary` to the new service.
    - Keep the handler focused on transport concerns, dependency coordination, and error handling.
- **Constraints**: Ensure the summarized output remains backward-compatible with existing MCP tool consumers.

## Execution Steps
1. Create `src/domain/sessions/activity-summary.ts`.
2. Move all `to*Summary` methods and related constants (like `ACTIVITY_PREVIEW_CHAR_LIMIT`) to the new service.
3. Update `CoreToolHandler` to accept the `ActivitySummaryService` as a dependency.
4. Refactor handler methods to call the summary service for formatting responses.
5. Add unit tests for `ActivitySummaryService` to verify formatting logic in isolation.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/domain/sessions/activity-summary.test.ts` (create if missing).
- **Criteria**: MCP tool responses for list/get operations remain identical in structure and content.
