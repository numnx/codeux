# Task Specification: T14 - Sprint 1

## Objective
Implement a standardized polling utility in `src/shared/polling/wait-until.ts` and migrate session wait loops in `CoreToolHandler` to use it.

## Files to Modify
- `src/shared/polling/wait-until.ts`: Create this new file.
- `src/mcp/core-tool-handler.ts`: Refactor `handleWaitForSessionCompletion`.

## Technical Details & Research Findings
- **Context**: Polling logic for session completion is currently manually implemented with `while` loops and `setTimeout` in `CoreToolHandler.ts`.
- **Utility Design**:
    - `waitUntil<T>(options: WaitOptions<T>): Promise<T>`
    - `WaitOptions` should include `predicate`, `action`, `intervalMs`, `timeoutMs`, and `onTimeout`.
    - Support for aborting the poll (e.g., via `AbortSignal`).
- **Constraints**: Maintain the existing logic that checks for `COMPLETED`, `FAILED`, `ACTION_REQUIRED`, or existence of a pull request.

## Execution Steps
1. Create `src/shared/polling/wait-until.ts`.
2. Implement the `waitUntil` helper with full type safety and error handling.
3. Refactor `CoreToolHandler.handleWaitForSessionCompletion` to use the new utility.
4. Add unit tests for `waitUntil` covering success, timeout, and predicate failure scenarios.
5. Verify that MCP tool polling still works correctly.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/shared/polling/wait-until.test.ts` (create if missing).
- **Criteria**: The utility correctly handles timeouts and resolves as soon as the predicate is met.
