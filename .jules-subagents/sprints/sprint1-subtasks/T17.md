# Task Specification: T17 - Sprint 1

## Objective
Extract the main merge CI feedback rendering logic from `SprintOrchestrator` into a dedicated domain service `src/domain/sprint/ci/main-merge-gate.ts`.

## Files to Modify

- **Context**: `renderMainMergeCiFeedback` handles the logic for providing feedback on the final merge PR status, including CI checks and blocking conditions.
- **Decomposition Target**:

    - Move logic for formatting the final merge feedback report.
    - Provide a clean interface: `renderMergeFeedback(context: MergeFeedbackContext): Promise<string>`.
- **Constraints**: Service should be purely logic-based and receive necessary state as input.

2. Move `renderMainMergeCiFeedback` logic to the new service.
3. Update `SprintOrchestrator` to call `MainMergeGateService.renderMergeFeedback` during the completion phase.
4. Add unit tests for `MainMergeGateService` covering various CI status scenarios for the main branch merge.
