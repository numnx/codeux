# Task Specification: T03 - Sprint 1

## Objective
Implement shared configuration value readers in `src/shared/config/value-readers.ts` to standardize how primitive types (port, boolean, integer, string) are parsed and validated across the codebase.

## Files to Modify
- `src/shared/config/value-readers.ts`: Create this new file.
- `src/repositories/settings-sanitizer.ts`: Source of existing logic.
- `src/config/app-config.ts`: Source of `readPortValue`.

## Technical Details & Research Findings
- **Context**: Port, integer, and boolean parsing is currently implemented locally in `settings-sanitizer.ts` and `app-config.ts`.
- **Logic to Migrate**:
    - `readPort(value: unknown, fallback: number): number`
    - `readBoolean(value: unknown, fallback: boolean): boolean`
    - `readInteger(value: unknown, fallback: number): number`
    - `readString(value: unknown, fallback: string): string`
- **Constraints**: Maintain existing validation rules (e.g., port must be 1-65535, integer must be finite).

## Execution Steps
1. Create `src/shared/config/value-readers.ts`.
2. Extract and refine parsing helpers from `src/repositories/settings-sanitizer.ts` and `src/config/app-config.ts`.
3. Ensure the helpers handle both string and primitive inputs where appropriate (especially for environment variables).
4. Add unit tests for each reader in `tests/backend/shared/config/value-readers.test.ts`.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/shared/config/value-readers.test.ts`.
- **Criteria**: All readers correctly parse valid inputs and return the fallback for invalid inputs.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
