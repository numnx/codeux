# Task Specification: T06 - Sprint 1

## Objective
Introduce `src/infrastructure/repositories/file-template-repository.ts` as a generic file-based lookup implementation and migrate `GuideRepository` and `InstructionRepository` to use it.

## Files to Modify
- `src/infrastructure/repositories/file-template-repository.ts`: Create this new file.
- `src/repositories/guide-repository.ts`: Refactor to use the new base repository.
- `src/repositories/instruction-template-repository.ts`: Refactor to use the new base repository.

## Technical Details & Research Findings
- **Context**: `GuideRepository` and `InstructionRepository` have nearly identical path building and file reading logic.
- **Generic Base Implementation**:
    - `FileTemplateRepository` should take a list of relative search directories (e.g., `['.jules-subagents/agents', '.jules-subagents/instructions']`).
    - Use `buildSearchRoots` from `src/shared/config/search-paths.ts`.
    - Implement a common `loadFile(name: string, repoPath?: string): Promise<string>` method.
- **Constraints**: Maintain current search behavior, including the typo-tolerant candidate list for instructions (e.g., "intructions").

## Execution Steps
1. Create `src/infrastructure/repositories/file-template-repository.ts`.
2. Implement the base lookup logic using `src/shared/config/search-paths.ts`.
3. Update `GuideRepository` to extend or wrap `FileTemplateRepository` for `.jules-subagents/agents`.
4. Update `InstructionRepository` to use it for `.jules-subagents/instructions` and `.jules-subagents/intructions`.
5. Verify with unit tests in `tests/backend/infrastructure/repositories/file-template-repository.test.ts`.

## Verification Requirements
- **Automated Tests**: Run `npm test` for the new repository tests and existing repository tests.
- **Criteria**: Guides and instruction templates are correctly located across all candidate paths.
