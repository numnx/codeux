# Task Specification: T30 - Sprint 1

## Objective
Refactor the test suite architecture by splitting monolithic test files, implementing shared test builders, and introducing comprehensive CI quality gate scripts.

## Files to Modify
- `package.json`: Add new test and quality gate scripts.
- `tests/backend/`: Substantially reorganize and refactor.
- `vitest.config.ts`: Update test configuration if necessary.

## Technical Details & Research Findings
- **Context**: The current test suite contains several oversized monolithic files (e.g., `sprint-orchestrator.test.ts`, `git-status-service.test.ts`) that are difficult to maintain and extend. CI scripts are also limited.
- **Improvements**:
    - Split mega-tests into domain-specific test files.
    - Introduce shared test builders (e.g., `buildMockSession`, `buildMockSubtask`) to reduce boilerplate.
    - Add `lint`, `typecheck`, `test:coverage`, and `ci` (combination of all checks) scripts.
- **Constraints**: Maintain current test coverage and ensure all existing tests pass after reorganization.

## Execution Steps
1. Reorganize `tests/backend/` to match the new `src/` structure.
2. Decompose `tests/backend/sprint/sprint-orchestrator.test.ts` and `tests/backend/services/git-status-service.test.ts`.
3. Create a `tests/builders/` directory for shared test data generators.
4. Update `package.json` with new scripts: `lint`, `typecheck`, `test:coverage`, `ci`.
5. Configure `vitest` to enforce a coverage threshold (e.g., 80% for critical modules).
6. Verify that the new `npm run ci` script passes all quality gates.

## Verification Requirements
- **Automated Tests**: Run `npm run ci`.
- **Criteria**: All tests pass, linting and typechecking pass, and coverage thresholds are met.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
