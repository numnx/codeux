# Task Specification: T09 - Sprint 1

## Objective
Add `/health` and `/ready` HTTP endpoints to the dashboard server to provide operational visibility and automated health checking for the MCP server.

## Files to Modify
- `src/server/dashboard-server.ts`: Add new route handlers.

## Technical Details & Research Findings
- **Context**: The server currently lacks standard health check endpoints.
- **Endpoint Specification**:
    - `/health`: Returns 200 OK with `{"status": "UP"}`. Indicates the process is alive.
    - `/ready`: Returns 200 OK only if the server is fully initialized (e.g., settings loaded, Jules API key checked). Otherwise, returns 503 Service Unavailable.
- **Readiness Criteria**:
    - Settings repository is initialized.
    - (Optional) Jules API connectivity check.

## Execution Steps
1. Add `/health` and `/ready` GET routes in `src/server/dashboard-server.ts`.
2. Implement a simple readiness check logic (can be passed via `DashboardServerOptions`).
3. Ensure the endpoints return deterministic JSON responses.
4. Verify by calling the endpoints locally using `curl` or a test script.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/server/dashboard-server.test.ts`.
- **Criteria**: `/health` and `/ready` return 200 OK after the server has started successfully.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
