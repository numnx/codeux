# Task Specification: T05 - Sprint 1

## Objective
Refactor `src/config/external-settings.ts` to use the shared search path resolver and value readers, and implement a reusable key normalization map for settings.

## Files to Modify
- `src/config/external-settings.ts`: Refactor `loadExternalSettingsHints` and `readSettingsJson`.

## Technical Details & Research Findings
- **Context**: `external-settings.ts` duplicates search path logic and manually maps multiple variants of environment and JSON keys (e.g., `JULES_API_KEY`, `JULES_KEY`).
- **Logic to Improve**:
    - Use `buildCandidatePaths` from `src/shared/config/search-paths.ts` instead of local `getSearchPaths`.
    - Create a canonical key normalization map for providers (Jules, Gemini, Codex, ClaudeCode, GitHub).
    - Use readers from `src/shared/config/value-readers.ts`.
- **Constraints**: Maintain current resolution logic for all providers and their key variants.

## Execution Steps
1. Update `src/config/external-settings.ts` imports.
2. Replace local `getSearchPaths` with `buildCandidatePaths`.
3. Implement a private `normalizeKey` helper or map to handle aliases for both Env and JSON sources.
4. Simplify `loadExternalSettingsHints` by iterating over a defined provider list.
5. Verify behavior with existing or new tests in `tests/backend/config/external-settings.test.ts`.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/config/external-settings.test.ts` (create if missing).
- **Criteria**: All provider keys are correctly resolved from environment variables and `settings.json` variants.
