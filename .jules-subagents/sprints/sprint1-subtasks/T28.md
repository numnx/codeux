# Task Specification: T28 - Sprint 1

## Objective
Decompose oversized dashboard components (`TaskCard`, `AiProviderSection`, `CliWorkflowSection`) into smaller, focused subcomponents to improve readability and maintainability.

## Files to Modify
- `dashboard/src/components/TaskCard.tsx`: Split into subcomponents.
- `dashboard/src/components/settings/AiProviderSection.tsx`: Split into subcomponents.
- `dashboard/src/components/settings/CliWorkflowSection.tsx`: Split into subcomponents.
- `dashboard/src/components/ui/`: Create new subcomponent files.

## Technical Details & Research Findings
- **Context**: Several key dashboard components have grown beyond 200 lines and handle multiple visual concerns (status badges, session feeds, provider configurations, Docker mounts).
- **Decomposition Target**:
    - `TaskCard`: Extract `StatusBadge`, `SessionFeed`, `TaskHeader`, `TaskMetadata`.
    - `AiProviderSection`: Extract `ProviderConfigRow`, `StrategySelector`.
    - `CliWorkflowSection`: Extract `DockerCredentialsSection`, `ExecutionModeSelector`.
- **Constraints**: Maintain the exact same visual appearance and interaction behavior using Tailwind CSS classes.

## Execution Steps
1. Identify natural boundaries for subcomponents within the target files.
2. Create new component files in `dashboard/src/components/` or a dedicated `ui/` folder.
3. Move JSX and related logic (e.g., `getStatusColor`) to the new components.
4. Update the parent components to use the new subcomponents.
5. Verify that the dashboard's look and feel remain unchanged.

## Verification Requirements
- **Manual Check**: Thoroughly review the dashboard UI to ensure no regressions in styling, layout, or interactivity for the refactored components.
- **Criteria**: Each individual component file is significantly reduced in size and complexity.

- Use the feature branch: feature/sprint1-implementation
