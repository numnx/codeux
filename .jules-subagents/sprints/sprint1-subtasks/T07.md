# Task Specification: T07 - Sprint 1

## Objective
Extract the service instantiation logic from the `JulesAgentServer` constructor into a dedicated `src/app/dependency-factory.ts` to improve testability and separate configuration from lifecycle management.

## Files to Modify
- `src/app/dependency-factory.ts`: Create this new file.
- `src/server/jules-agent-server.ts`: Refactor the constructor and class properties.

## Technical Details & Research Findings
- **Context**: The `JulesAgentServer` constructor currently wires up over 15 services and repositories inline, making it difficult to test in isolation.
- **Dependency Factory Design**:
    - Define a `RuntimeDependencies` interface containing all instantiated services.
    - Implement `createRuntimeDependencies(options: JulesAgentServerOptions): RuntimeDependencies`.
    - Handle the complex cross-dependencies (e.g., `TaskService` needing `CliWorkflowService` and `JulesApiClient`).
- **Constraints**: Maintain the exact same wiring and singleton behavior as the current constructor.

## Execution Steps
1. Create `src/app/dependency-factory.ts`.
2. Define the `RuntimeDependencies` interface based on the properties in `JulesAgentServer`.
3. Move the instantiation logic from the `JulesAgentServer` constructor to `createRuntimeDependencies`.
4. Refactor `JulesAgentServer` to accept `RuntimeDependencies` in its constructor (or a factory result).
5. Ensure all existing functionality (MCP tools, dashboard) still works correctly after the handover.

## Verification Requirements
- **Automated Tests**: Run the full backend test suite (`npm test`) to ensure no regressions in service wiring.
- **Criteria**: `JulesAgentServer` constructor is reduced to simple property assignment and route registration.

## Engineering Standard
- Use the feature branch: feature/sprint1-implementation
- Ensure all tests pass before completing.
