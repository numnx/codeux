# Task Specification: T02 - Sprint 1

## Objective
Implement a shared search path resolver in `src/shared/config/search-paths.ts` to centralize the logic for finding configuration files and guides across the project.

## Files to Modify
- `src/shared/config/search-paths.ts`: Create this new file.

## Technical Details & Research Findings
- **Context**: Path resolution logic is currently duplicated in `app-config.ts`, `external-settings.ts`, and `guide-repository.ts`. They all use a similar pattern of checking `process.cwd()`, `projectRoot`, and `os.homedir()`.
- **Logic**:
    - `buildSearchRoots(projectRoot: string, repoPath?: string): string[]`: Returns a unique list of base directories to search in (repoPath, cwd, projectRoot, homedir).
    - `buildCandidatePaths(relativePath: string, projectRoot: string, repoPath?: string): string[]`: Combines roots with the relative path.
- **Constraints**: Must handle deduplication and maintain the established precedence (Repo > CWD > Project > Home).

## Execution Steps
1. Create `src/shared/config/search-paths.ts`.
2. Implement `buildSearchRoots` and `buildCandidatePaths` with proper typing.
3. Add unit tests for these helpers to ensure precedence and deduplication work as expected.
4. Verify that the new helpers can replace existing logic in `GuideRepository` and `AppConfig`.

## Verification Requirements

- **Criteria**: Tests pass with various combinations of project root and repo path.
