# Task Specification: T12 - Sprint 1

## Objective
Refactor `src/integrations/jules-api-client.ts` to eliminate `any` usage in API calls and introduce strict request/response interfaces for all Jules API methods.

## Files to Modify
- `src/integrations/jules-api-client.ts`: Refactor method signatures and internal logic.

## Technical Details & Research Findings
- **Context**: The current `JulesApiClient` implementation uses `any` for request parameters and some response types (e.g., `listSources`, `createSession`), which reduces type safety at the boundary.
- **Improvements**:
    - Define explicit `Request` and `Response` interfaces for `getSource`, `listSources`, `createSession`, etc.
    - Normalize session name and ID handling using a consistent internal helper.
    - Ensure all `axios` calls use generic type parameters for responses.
- **Constraints**: Maintain compatibility with the existing `JulesSession`, `JulesActivity`, and `JulesSource` types defined in `app-types.ts`.

## Execution Steps
1. Define strict interfaces for all Jules API request payloads and response bodies.
2. Update all methods in `JulesApiClient` to use these new interfaces instead of `any`.
3. Refactor `listSources`, `listSessions`, and `listActivities` to use typed parameter objects.
4. Ensure `createSession` takes a strongly typed input object.
5. Verify that all calls to `JulesApiClient` in the codebase are updated to match the new strict signatures.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/services/jules-api-client.test.ts` (create if missing).
- **Criteria**: TypeScript compiler passes without `any` escape hatches in the client.
