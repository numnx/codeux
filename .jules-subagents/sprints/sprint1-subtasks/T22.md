# Task Specification: T22 - Sprint 1

## Objective
Substantially refactor `src/services/cli-workflow-service.ts` by decomposing it into focused modules: `workspace-manager`, `provider-runner`, `docker-runner`, and `pr-service`.

## Files to Modify
- `src/services/cli-workflow-service.ts`: Decompose and refactor.
- `src/infrastructure/providers/cli/workspace-manager.ts`: Create new module.
- `src/infrastructure/providers/cli/provider-runner.ts`: Create new module.
- `src/infrastructure/providers/cli/docker-runner.ts`: Create new module.
- `src/infrastructure/providers/cli/pr-service.ts`: Create new module.

## Technical Details & Research Findings
- **Context**: The `CliWorkflowService` is the largest file in the project (~1300 lines) and mixes many unrelated responsibilities.
- **Decomposition Plan**:
    - `WorkspaceManager`: Handles worktree creation, cleanup, and path resolution.
    - `ProviderRunner`: Handles provider-specific command construction (Gemini, Codex, ClaudeCode) and execution.
    - `DockerRunner`: Handles Docker-specific execution logic, mount mapping, and container lifecycle.
    - `PrService`: Handles PR creation, branch management, and git push operations.
- **Constraints**: Preserve all existing behaviors, including Docker mount mapping, retry logic, and session activity logging.

## Execution Steps
1. Create the new module files in `src/infrastructure/providers/cli/`.
2. Move logic from `CliWorkflowService` into the respective new modules.
3. Define clear interfaces for each module to facilitate dependency injection.
4. Update `CliWorkflowService` to act as a high-level orchestrator that delegates to these specialized modules.
5. Ensure that all session activity logging is preserved during the transition.

## Verification Requirements
- **Automated Tests**: Run `npm test tests/backend/services/cli-workflow-service.test.ts`.
- **Criteria**: All existing CLI workflow tests pass, and the file size of `CliWorkflowService` is reduced below 200 lines.
