# Task Specification: T17 - Sprint 2

## Objective
Improve session sync performance by deduping activity fetches and introducing bounded parallel retrieval for matched sessions.

## Files to Modify
- `src/sprint/steps/session-sync-step.ts`: Refactor activity retrieval and task update flow.
- `tests/backend/sprint/session-sync-step.test.ts`: Add performance-logic and dedupe coverage.

## Technical Details & Research Findings
- **Context**: current implementation may perform repeated per-task activity fetch work.
- **Goal**: maintain O(tasks + sessions) matching and efficient activity retrieval.
- **Constraint**: keep deterministic task-session binding by run key.

## Execution Steps
1. Build session map once and collect unique session names for matched tasks.
2. Fetch recent activities using bounded parallelism (not unbounded Promise fan-out).
3. Map fetched activities back to tasks deterministically.
4. Add test cases for collision handling and deduped activity fetch counts.

## Verification Requirements
- **Automated Tests**: `npm run test -- tests/backend/sprint/session-sync-step.test.ts`; `npm run typecheck`
- **Criteria**: Activity fetches are deduped per matched session.
- **Criteria**: Session sync behavior remains deterministic and scalable.
