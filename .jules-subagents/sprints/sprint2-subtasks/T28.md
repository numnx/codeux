## Objective
Reduce repetitive dashboard settings component code by introducing descriptor-driven field rendering for common setting controls.

- `dashboard/src/components/settings/CiIntelligenceSection.tsx`: Refactor to schema-driven rendering.

## Technical Details & Research Findings
- **Context**: multiple settings sections still contain long repeated UI and update logic blocks.
- **Goal**: centralize control definitions for readability, maintainability, and consistency.
- **Constraint**: keep current visual behavior, accessibility, and save semantics unchanged.

2. Implement reusable settings field renderer for descriptor-driven UI.
3. Migrate targeted settings sections and provider/credential rows to descriptor model.
4. Add/update tests for immutability and expected update behavior.
