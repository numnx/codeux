# Task Specification: T07 - Sprint 2

## Objective
Create a typed runtime context object and migrate mutable server state access/update logic out of ad-hoc closures.

## Files to Modify
- `src/app/runtime-context.ts`: Create runtime state container and typed access APIs.
- `src/server/jules-agent-server.ts`: Replace closure-based context with runtime context usage.
- `src/app/dependency-factory.ts`: Consume runtime context adapters.

## Technical Details & Research Findings
- **Context**: server currently exports mutable state via broad closure surface in `createContext`.
- **Goal**: reduce coupling and make state transitions explicit and testable.
- **Constraint**: avoid changing observable behavior of status/session/cache updates.

## Execution Steps
1. Define runtime context interface for last status, failures, settings mirrors, and runtime ports.
2. Refactor server and factory to use runtime context getters/setters.
3. Eliminate anonymous mutable closure fields replaced by typed methods.
4. Add targeted tests for context state transitions where applicable.

## Verification Requirements
- **Automated Tests**: `npm run typecheck`; `npm run test -- tests/backend/smoke.test.ts`
- **Criteria**: State access in server/factory is context-driven and typed.
- **Criteria**: No regression in orchestration/dashboard runtime status behavior.

## Engineering Standard
- Use the feature branch: feature/sprint2-implementation
- Ensure all tests pass before completing.
