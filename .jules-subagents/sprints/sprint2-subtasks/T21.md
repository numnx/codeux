# Task Specification: T21 - Sprint 2

## Objective
Decompose `GitStatusService` so command queries, JSON mapping, and tracking policy are separated into focused modules.

## Files to Modify
- `src/services/git-status-service.ts`: Reduce to orchestration layer.
- `src/infrastructure/git/git-status-query-client.ts`: Create command query module.
- `src/infrastructure/git/git-status-mappers.ts`: Create response mapping module.
- `src/infrastructure/git/git-status-policy.ts`: Create tracking/policy helpers.

## Technical Details & Research Findings
- **Context**: current service mixes execution, mapping, enrichment, and policy in one class.
- **Goal**: improve maintainability and unit-test granularity.
- **Constraint**: preserve external `GitStatusService` API surface and behavior.

## Execution Steps
1. Extract gh/git command query logic into query client.
2. Extract JSON-to-contract mapping into dedicated mappers.
3. Extract tracking and filter policy logic into reusable policy module.
4. Keep service class as orchestration and compatibility wrapper.

## Verification Requirements
- **Automated Tests**: `npm run test -- tests/backend/services/git-status-service.test.ts`; `npm run typecheck`
- **Criteria**: Git status orchestration no longer embeds all query/mapping details.
- **Criteria**: Service behavior remains compatible and fully tested.

## Engineering Standard
- Use the feature branch: feature/sprint2-implementation
- Ensure all tests pass before completing.
