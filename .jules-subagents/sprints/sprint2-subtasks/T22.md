# Task Specification: T22 - Sprint 2

## Objective
Improve git status runtime performance by parallelizing base GH queries and bounding failed-run/job enrichment concurrency.

## Files to Modify
- `src/services/git-status-service.ts`: Apply parallel fetch orchestration and bounded enrichment.
- `tests/backend/services/git-status-service.test.ts`: Add concurrency behavior assertions.

## Technical Details & Research Findings
- **Context**: PR list, CI runs, and merged PR queries currently execute sequentially.
- **Goal**: reduce status latency while controlling API/resource usage.
- **Constraint**: avoid unbounded concurrent GH calls and preserve deterministic output.

## Execution Steps
1. Use `Promise.all` for independent base queries (PRs, runs, merged PRs).
2. Implement bounded concurrency for failed-run job/log enrichment.
3. Preserve warning accumulation and sorting semantics.
4. Add tests validating unchanged data interpretation with new execution order.

## Verification Requirements
- **Automated Tests**: `npm run test -- tests/backend/services/git-status-service.test.ts`; `npm run typecheck`
- **Criteria**: Base GH status queries run in parallel.
- **Criteria**: Enrichment concurrency is bounded and behavior-compatible.
