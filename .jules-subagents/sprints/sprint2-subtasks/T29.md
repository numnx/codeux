# Task Specification: T29 - Sprint 2

## Objective
Replace independent runtime/git polling loops with a single coordinated scheduler managing refresh cadence, backoff, and error state.

## Files to Modify
- `dashboard/src/hooks/use-dashboard-runtime-data.ts`: Consume unified poll coordinator.
- `dashboard/src/hooks/use-dashboard-poll-manager.ts`: Refactor into scheduler with fan-out targets.
- `dashboard/src/lib/api/dashboard-api.ts`: Keep endpoint calls compatible with coordinated polling flow.
- `tests/dashboard/lib/status.test.ts`: Update if runtime stats merge behavior changes.

## Technical Details & Research Findings
- **Context**: current UI uses two independent poll loops with duplicated timing/backoff logic.
- **Goal**: central orchestration of status and git refresh to reduce drift/race conditions.
- **Constraint**: preserve manual refresh behavior and visible error messaging semantics.

## Execution Steps
1. Define unified poll scheduler state for pending/error/backoff metadata.
2. Refactor runtime hook to run status+git refreshes via one scheduler entry point.
3. Ensure manual refresh APIs still target correct refresh operations.
4. Validate no regressions in task activity merge and stats derivation.
