# Task Specification: T02 - Sprint 2

## Objective
Eliminate `any` from core runtime contracts in `src/contracts/app-types.ts` by introducing typed interfaces for session outputs, activity payloads, and runtime settings shapes.

## Files to Modify
- `src/contracts/app-types.ts`: Replace `any`-based fields and index signatures with strict types.
- `tests/backend/domain/sessions/activity-summary.test.ts`: Update coverage for new typed activity/session payload variants.
- `tests/backend/mcp/core-tool-handler.test.ts`: Adjust mocks to satisfy stricter contract types.

## Technical Details & Research Findings
- **Context**: Contract-level `any` creates unsafe assumptions in MCP handlers, orchestration, and summary formatters.
- **Refactor target**: `JulesSession.outputs`, `JulesActivity` extension fields, and `Settings` shape.
- **Constraint**: Preserve runtime payload compatibility while tightening compile-time guarantees.

## Execution Steps
1. Define explicit interfaces/unions for known output payload shapes used by runtime code.
2. Replace open-ended `any` fields with `unknown` + typed helpers or concrete structured types.
3. Update downstream callsites/tests to use the new contracts safely.
4. Confirm no behavior change in serialization/summary output.
