# Task Specification: T11 - Sprint 2

## Objective
Introduce runtime argument validation before tool handler invocation so malformed inputs fail fast with deterministic MCP errors.

## Files to Modify
- `src/api/mcp/validators/`: Create per-tool validator modules.
- `src/server/mcp-request-router.ts`: Execute validators before dispatch.
- `tests/backend/mcp/core-tool-handler.test.ts`: Add malformed payload tests.
- `tests/backend/mcp/agent-tool-handler.test.ts`: Add malformed payload tests.

## Technical Details & Research Findings
- **Context**: handler layer currently trusts incoming MCP arguments.
- **Goal**: enforce schema-compatible payloads at boundary to reduce runtime failure ambiguity.
- **Constraint**: return MCP-friendly error shape and preserve current successful behavior.

## Execution Steps
1. Define runtime validators for each tool argument payload.
2. Integrate validation pipeline into request router before handler dispatch.
3. Map validation failures to clear MCP error responses with path-aware messages.
4. Add tests for validation fail/pass scenarios.

## Verification Requirements
- **Automated Tests**: `npm run test -- tests/backend/mcp/core-tool-handler.test.ts`; `npm run test -- tests/backend/mcp/agent-tool-handler.test.ts`; `npm run typecheck`
- **Criteria**: Invalid tool arguments are rejected before business logic execution.
- **Criteria**: Validation failures are deterministic and human-readable.

## Engineering Standard
- Use the feature branch: feature/sprint2-implementation
- Ensure all tests pass before completing.
