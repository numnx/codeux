# Task Specification: T15 - Sprint 2

## Objective
Convert watch loop flow into explicit typed state transitions to improve clarity, observability, and future maintenance.

## Files to Modify
- `src/domain/sprint/orchestrator/watch-loop-runner.ts`: Replace implicit while-loop branching with state-machine transitions.
- `src/domain/sprint/orchestrator/watch-loop-state-machine.ts`: Create transition model module.
- `tests/backend/sprint/watch-loop-core.test.ts`: Expand transition coverage.

## Technical Details & Research Findings
- **Context**: current watch loop method mixes report rendering, completion handling, and loop control in one block.
- **Goal**: explicit states (`RUNNING`, `CHECKPOINT`, `FINISHED`, etc.) and deterministic transitions.
- **Constraint**: output behavior and cleanup semantics must remain compatible.

## Execution Steps
1. Define state enum and transition rules based on current loop behavior.
2. Refactor loop execution to process transitions instead of nested conditionals.
3. Move completion/cleanup handlers into explicit transition handlers.
4. Add tests for each terminal and checkpoint transition path.

## Verification Requirements
- **Automated Tests**: `npm run test -- tests/backend/sprint/watch-loop-core.test.ts`; `npm run typecheck`
- **Criteria**: Watch loop transitions are explicit and type-safe.
- **Criteria**: Existing completion/checkpoint behavior remains intact.
