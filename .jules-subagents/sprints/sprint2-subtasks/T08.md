# Task Specification: T08 - Sprint 2

## Objective
Refactor `JulesAgentServer` so lifecycle concerns (settings boot, dashboard boot, MCP boot, recovery) are handled by focused services/modules.

## Files to Modify
- `src/server/jules-agent-server.ts`: Reduce to lifecycle coordinator.
- `src/app/lifecycle/settings-lifecycle-service.ts`: Create module.
- `src/app/lifecycle/dashboard-lifecycle-service.ts`: Create module.
- `src/app/lifecycle/mcp-lifecycle-service.ts`: Create module.

## Technical Details & Research Findings
- **Context**: server class still handles many responsibilities in one unit.
- **Goal**: isolate lifecycle steps for maintainability and startup diagnostics.
- **Constraint**: preserve startup ordering and failure semantics.

## Execution Steps
1. Extract settings loading/sync logic into lifecycle service.
2. Extract dashboard setup and runtime-port capture into dedicated service.
3. Extract MCP transport startup/teardown setup into dedicated service.
4. Keep server class as coordinator with clear stage boundaries.
