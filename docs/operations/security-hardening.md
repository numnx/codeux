# Security Hardening

This document outlines the concrete security protections implemented in Code UX to protect the single-user trusted process against unintended cross-origin misuse and path traversal attacks.

## Dashboard Protections

### REST API Defense
- **Origin and Fetch-Metadata Checks**: All mutating operations and sensitive reads enforce strict `Origin` validation and leverage Sec-Fetch-Site metadata (e.g., rejecting cross-site requests).
- **Security Headers**: Standard security headers (such as `X-Content-Type-Options: nosniff`, `Content-Security-Policy`, and appropriate cache controls) are enforced on HTTP responses.

### WebSocket Connections
- **WebSocket Origin Validation**: The `/ws` upgrade handlers perform strict Origin validation. Cross-origin WebSocket connections from untrusted domains are explicitly rejected.

## MCP Gateway Hardening

- **HTTP Bearer & Session Hardening**: When bound beyond loopback, the MCP HTTP gateway requires a bearer token (`--mcp-http-auth-token`). It correctly isolates session boundaries and ignores standard cookies to prevent CSRF-style misuse.

## Data and File Constraints

### Sprint Preview and File Browser
- **Path Constraints**: The file browser and preview server use resolved, absolute path containment checks to strictly prevent escaping the designated workspace using `../` traversal or symbolic link trickery.
- **Proxy Constraints**: The sprint preview proxy prevents server-side request forgery (SSRF) and bounds traffic to legitimate development ports, refusing to proxy administrative or system-level local destinations.

### Ingestion Limits
- **Upload/Path Ingestion Limits**: File ingestion APIs enforce maximum file size limits (preventing memory exhaustion via large payloads) and perform strict path sanitization on the uploaded filenames or ingestion source targets.
- **Markdown URL Sanitization**: Markdown generated by workers or rendered in the dashboard sanitizes `javascript:` URIs and other hostile link targets to prevent persistent cross-site scripting (XSS) if clicked.

## Secret Redaction

- Log output filters and redacts known API keys (e.g., `JULES_API_KEY`) and other sensitive values before they are written to disk or sent to the dashboard logging stream.
