# Managing Projects

Learn how to effectively manage your projects by handling lifecycle operations like archiving and restoring, utilizing Project Templates, and setting project visibility.

## Before you start

* Ensure you have access to a Workspace.
* You need appropriate permissions to archive, restore, and change the visibility of projects.

## Project Visibility: Private vs. Workspace-visible

Understanding project visibility is critical to ensuring the right team members have access to your work.

* **Private**: A private project is only accessible to members who are explicitly invited to it. It is not visible in the general workspace directory. This is ideal for sensitive work, drafts, or highly restricted team projects.
* **Workspace-visible**: A workspace-visible project is accessible to all members within the workspace. Anyone in the workspace can find, view, and join the project. This is recommended for general team collaboration and transparent workflows.

## Steps

### Archiving a Project

Archiving a project removes it from active views without permanently deleting the data.

1. Navigate to the **Projects** directory in your workspace sidebar.
2. Click on the project you want to archive to open its dashboard.
3. Click the **Settings** gear icon in the top right corner.
4. Scroll down to the **Danger Zone** section.
5. Click the **Archive Project** button.
6. Confirm the action by clicking **Archive** in the confirmation dialog.

### Restoring an Archived Project

If you need to resume work on an archived project, you can easily restore it to an active state.

1. Navigate to the **Projects** directory in your workspace sidebar.
2. Click the **Filter** dropdown menu at the top of the page.
3. Select **Archived** to view all archived projects.
4. Click on the archived project you want to restore.
5. Click the **Restore Project** button displayed in the banner at the top of the project view.
6. Confirm the action to move the project back to the active list.

### Using and Managing Project Templates

Project Templates allow you to standardize workflows and quickly spin up new projects with predefined tasks and structures.

1. Navigate to the **Projects** directory and click the **New Project** button.
2. Choose **Use a Template** instead of starting from scratch.
3. Browse the available templates and click **Select** on the one that fits your needs.
4. To create a new template from an existing project, go to the project's **Settings**, scroll to **Template Management**, and click **Save as Template**.
5. To manage existing templates, navigate to the **Workspace Settings**, click on **Templates**, and use the edit or delete options next to each template to keep your library up to date.

## Expected result

You should be able to confidently set project visibility (Private or Workspace-visible), archive and restore projects as their lifecycle dictates, and leverage Project Templates to streamline new project creation.
