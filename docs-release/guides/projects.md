# Managing Projects

## Overview
This guide covers the lifecycle management of Projects, including archiving, restoring, templates, and visibility. You will learn how to organize your workspace effectively by utilizing these features.

::: info
Archiving a project hides it from the active workspace views but retains all of its data. It can be restored at any time.
:::

## Prerequisites
* You must have an active Workspace with "Manager" or "Admin" permissions to manage templates and archive/restore projects.

## Steps

### Understanding Project Visibility
Projects can have two levels of visibility:
* **Private**: The project is only visible to you and any members you explicitly invite. It will not appear in the workspace directory for uninvited users.
* **Workspace-visible**: The project is accessible to all members of the workspace. Anyone can find it in the directory and join it.

### 1. Archiving a Project
Archiving removes the project from active lists to keep your workspace uncluttered.

1. Navigate to the **Projects** tab.
2. Select the specific project you want to archive.
3. Click the **Settings** icon (gear) in the top right corner.
4. Scroll to the bottom and click **Archive Project**.
5. Confirm the action in the dialogue box by clicking **Archive**.

### 2. Restoring a Project
If you need to access an archived project, you can restore it to the active workspace.

1. Navigate to the **Projects** tab.
2. Click on the **Filter** dropdown and select **Show Archived**.
3. Locate the project you want to restore and click on it.
4. Click the **Restore Project** button at the top of the project dashboard.

### 3. Using Project Templates
Project Templates allow you to create new projects with pre-defined tasks, milestones, and settings.

1. Navigate to the **Projects** tab.
2. Click the **New Project** button.
3. Instead of "Blank Project", select **Use Template**.
4. Browse the available templates and select one that fits your needs.
5. Review the template details and click **Create from Template**.

### 4. Managing Project Templates
You can create your own templates or manage existing ones if you have the correct permissions.

1. Navigate to workspace **Settings**.
2. Select the **Templates** tab in the sidebar.
3. To create a new template, click **Create Template** and follow the prompts.
4. To edit or delete an existing template, click the three dots (`...`) next to the template name and choose the respective action.

## Expected Result
* **Visibility:** You should be able to set projects as Private or Workspace-visible.
* **Archiving/Restoring:** Archived projects should move to the "Archived" view, and restoring them should bring them back to the active list.
* **Templates:** Creating a project from a template should pre-populate the project with the template's structure.

::: tip
Regularly archiving completed or inactive projects helps maintain a clean and performant workspace for your team.
:::
