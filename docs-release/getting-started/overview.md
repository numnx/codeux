# Sprint OS Overview

## Overview
This guide provides a high-level introduction to Sprint OS. By the end of this page, you will understand what Sprint OS is, who it is designed for, and when you should use it in your workflow.

::: info
Sprint OS is a powerful orchestration tool that brings multi-provider virtual worker engines and DAG dependency scheduling to your local environment.
:::

## What is this?
Sprint OS is a production-grade Model Context Protocol (MCP) server and multi-provider virtual worker engine. It provides a suite of sprint orchestration tools paired with a real-time Preact dashboard to monitor and manage your development sprints.

## Who is it for?
This platform is designed for senior engineers, maintainers, and development teams who need an intelligent orchestration engine to automate complex development workflows with AI integrations.

## When should I use it?
Use Sprint OS when you need dependency-aware execution for development tasks, automated PR merge gates, or intelligent AI delegation using the supported virtual worker integrations.

## Core Capabilities

* **Sprint Orchestration**: Intelligent task delegation with built-in DAG dependency scheduling.
* **Live Web Dashboard**: A real-time V2 Preact dashboard for monitoring sprint progress locally.
* **AI Integrations**: Built-in support for five major AI providers (Jules, Gemini, Codex, Claude Code, and Qwen Code).
* **MCP Compliance**: Exposes enterprise-grade MCP tools and connects seamlessly via the Model Context Protocol.
